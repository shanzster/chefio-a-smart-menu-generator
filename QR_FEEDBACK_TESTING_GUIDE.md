# QR Feedback System - Testing Guide

You're trying to access `/feedback/53212` but this ID doesn't exist in your Firebase `qrCodes` collection. The ID "53212" is just a test/example ID.

## How to Test Properly

### Step 1: Clear Browser Cache
1. Open DevTools (F12)
2. Go to Application tab
3. Click "Clear storage"
4. Check all boxes
5. Click "Clear site data"
6. Refresh the page (Ctrl+Shift+R or Cmd+Shift+R)

### Step 2: Login as Cook
1. Go to `/login`
2. Login with your cook account

### Step 3: Save a Recipe (if you haven't already)
1. Go to `/cook/menu-generator`
2. Generate a recipe
3. Click "Save Recipe" button
4. Wait for success message

### Step 4: Generate QR Code
1. Go to `/cook/qr-generator`
2. You should see your saved recipes listed
3. Click on a recipe to select it
4. Click "Generate QR Code" button
5. Wait for the QR code to appear
6. The system will create a new entry in Firebase `qrCodes` collection
7. The QR code will have a REAL Firebase ID (not "53212")

### Step 5: Test Feedback Page
1. Copy the link shown below the QR code
2. Open it in a new tab or incognito window
3. You should see:
   - Dish image
   - Dish name and description
   - Cook name ("Prepared by...")
   - Servings
   - Nutritional information (calories, protein, carbs, fat)
   - Ingredients list
   - Instructions (if available)
   - Rating form
   - Feedback text area
   - Guest name input

### Step 6: Submit Feedback
1. Rate the dish (overall rating is required)
2. Optionally rate categories (taste, presentation, creativity, portion size)
3. Optionally add text feedback
4. Optionally enter your name
5. Click "Submit Feedback"
6. You should see a "Thank You" message

## Firebase Structure

```
qrCodes (collection)
├── {qrCodeId} (auto-generated by Firebase)
│   ├── recipeId: "abc123"
│   ├── recipeName: "Spicy Thai Chicken"
│   ├── recipeImage: "https://..."
│   ├── cookId: "user123"
│   ├── cookName: "John Doe"
│   ├── title: "Spicy Thai Chicken"
│   ├── description: "A delicious..."
│   ├── servings: 4
│   ├── ingredients: ["chicken", "sauce", ...]
│   ├── instructions: ["Step 1", "Step 2", ...]
│   ├── nutrition: { calories: "450", protein: "35g", ... }
│   ├── createdAt: timestamp
│   ├── scans: 0
│   ├── feedbackCount: 0
│   └── feedback (subcollection)
│       └── {feedbackId}
│           ├── ratings: { overall: 5, taste: 5, ... }
│           ├── feedback: "Great dish!"
│           ├── guestName: "Jane"
│           └── createdAt: timestamp
```

## Troubleshooting

### "Dish Not Found" Error
- You're using an invalid QR code ID
- Generate a new QR code from `/cook/qr-generator`
- Use the link provided by the system

### "No Saved Recipes" in QR Generator
- Go to Menu Generator first
- Generate and save a recipe
- Then return to QR Generator

### Service Worker Issues
- Clear browser cache completely
- Unregister service workers in DevTools
- Hard refresh (Ctrl+Shift+R)

### 500 Server Error
- Check browser console for detailed error
- Make sure Firebase is configured correctly
- Check `.env` file has correct Firebase credentials

## What Makes This System Flexible

The QR code system now supports recipes with:
- ✅ Different numbers of ingredients (some have 5, some have 20)
- ✅ Different numbers of instructions (some have 3 steps, some have 10)
- ✅ Optional nutrition information (shows only if available)
- ✅ Optional images (shows placeholder if missing)
- ✅ Optional descriptions
- ✅ Cook attribution (shows who made it)

The Feedback page dynamically displays whatever data is available in the QR code entry!
